# 🧠 Regentix

**Policy-driven MCP proxy for secure LLM tool execution using Rego-based governance and AI-generated rules.**

---

## 🚪 Overview

**Regentix** is a proxy layer for MCP (Model Context Protocol) servers that enforces fine-grained, **policy-driven access control** over LLM tool execution.

It acts as a **semantic governance gateway** between Claude Desktop (and other MCP-compatible clients) and external MCP servers.

Every tool call generated by an LLM is intercepted and validated against **Rego policies (Regorus engine)** before execution.

---

## 🧩 Key Features

- 🔐 Rego-based policy enforcement via **Regorus**
- 🤖 AI-generated policies using fine-tuned **Qwen2.5-Coder-1.5B-Instruct**
- 🧠 Synthetic dataset generation with **Google Gemini**
- 🚪 MCP proxy for **Claude Desktop integration**
- 🧾 Fine-grained access control (e.g., repository-level Git restrictions)
- 🌐 Web dashboard for policy creation and management
- ⚙️ Multi-language architecture (Rust + Python + Angular)
- 🛡️ Deny-by-default execution model

---

## 🏗️ Architecture

Regentix is composed of three main components:

### 🦀 MCP Proxy Server (Rust - STDIO)
- Entry point of the system
- Intercepts MCP tool calls from Claude Desktop
- Evaluates requests using **Regorus (Rego engine)**
- Forwards allowed requests to MCP servers

### 🐍 Backend API (Python)
- Provides API layer for frontend
- Runs AI policy generation pipeline
- Handles model inference and Rego generation

### 🌐 Web Dashboard (Angular)
- UI for creating and managing policies
- Natural language → Rego policy generation
- Policy testing and validation interface
